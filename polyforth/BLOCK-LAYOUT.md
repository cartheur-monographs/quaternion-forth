# Volatco Block Layout

This repo treats the Volatco project image as a `4800`-block `polyForth`
space.

For this project, the intended layout is:

```text
4800-block VOLATCO.src image

    0                                                            4799
    |---------------------------------------------------------------|

    0      9 10 11                                              30 31
    |------|--|--------------------------------------------------|--->
     reserved  load block      quaternion source blocks           empty
               10 LOAD          generated from volatco-blocks.pf

    block range     purpose
    -----------     ---------------------------------------------
    0..9            reserved
    10              quaternion project load block
    11..30          source blocks from polyforth/volatco-blocks.pf
    31..4799        currently unused by this repo
```

That choice follows the current working assumption that user application code
should begin after block `9`.

## Why block `10`

The saneForth examples use block `1585` as a demo load block, but that is part
of a different stock image layout. In the Volatco project source file
(`VOLATCO.src`), block `0` is just a project header and the rest of the file is
empty. That makes block `10` a clean project-specific entry point without
reusing the stock demo location.

## Load convention

After entering `AFORTH`, the intended command is:

```text
10 LOAD
```

That load block explicitly loads blocks `11` through `30`.

Then verify:

```text
qreport
mreport
```

## Image generation

The project image is generated by:

```bash
python3 tools/pf_block_pack.py \
  polyforth/volatco-blocks.pf \
  deploy/windows7-volatco/VOLATCO.src
```

The packer enforces a classical block shape of `16` lines by `64` characters
per block.
